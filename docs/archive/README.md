# 归档文档说明

本目录保存已经完成或被替换的历史资料，供回溯与比对使用。当前按照**月份**与 **legacy** 两个层级组织：

- `2025-10/`：2025 年 10 月的需求与问题诊断记录  
- `2025-11/`：2025 年 11 月的会议纪要、临时指引与提交说明  
- `legacy/`：早期流程文档、规范草稿等长期保留但不再更新的资料

## ⚠️ 使用前的快速检查
1. **确认日期**：只在需要追溯当时背景时使用过期文档。
2. **对照现状**：阅读完成后务必核对当前实现或规范。
3. **记录差异**：如发现现状与历史文档不一致，请在现行规范或知识库补充说明。

## 📁 目录索引

### 2025-10/
- `requirements/`：胶卷页面旧需求稿
- `问题诊断/`：懒加载、地图 API 等问题的诊断记录与总结

### 2025-11/
- `临时文档/`：2025-11 的会议纪要、图片旋转专项文档与提交指引

### legacy/
- `docs-整理方案.md` 等：早期的目录规划、流程草稿与检查清单
- `历史版本/`：旧版框架总结等历史材料

## 🔄 归档流程
1. 将准备归档的文件移动到对应月份或 `legacy/`。
2. 在文档头部标注归档日期与最新状态（例如“⚠️ 已过时，仅供参考”）。
3. 如出现全新类别，可在相应月份下新增子目录，并更新本索引。
